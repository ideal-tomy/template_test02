Next.js と Supabase の組み合わせは、スピード感のある開発と、AI 機能をエッジ（Edge Functions）で動かすアーキテクチャとして完璧な選択ですね。
ご提示いただいた「3×2＋α」の構成をベースに、開発のロードマップとしても使える全体像の要件定義書をまとめました。
1. システム概要
人材紹介・派遣業者が、候補者の管理から入管書類作成、収益管理までを一気通貫で行う「AI 搭載型業務管理コックピット」。
ターゲット: 外国人（主に特定技能・技能実習）を扱う人材エージェント
コンセプト: 「管理を便利に、判断をAIで」

Supabase (PostgreSQL) 上に以下のテーブルを構築する。Row Level Security (RLS) を有効にし、組織（管理会社）ごとにデータが分離されるように設計。
主要テーブル:
 * organizations: 管理会社情報
 * candidates: スリランカ人候補者（氏名、連絡先、日本語レベル、AIスコア、OCR抽出済みパスポート情報）

 * knowledge_base: 社内ナレッジ（ベクトルデータ格納 pgvector 使用）

① OCR・書類自動生成フロー

 * OpenAI Vision API が画像を解析し、氏名・番号・生年月日を JSON 形式で抽出。
 * candidates テーブルに自動保存し、react-pdf を用いて入管書類の対応箇所を埋める。

直感的な操作性と、情報の「引き出しやすさ」を最優先したデザイン要件定義書を作成しました。
ホワイトベースの清潔感ある管理画面を保ちつつ、スマホ特有の「限られた画面スペース」をボトムシートやフローティングボタンで解決する設計です。AIエージェントへの指示出しにそのままお使いください。
UI/UX デザイン要件定義書
1. デザインコンセプト： "Invisible Admin" (直感的な管理)
 * 視覚的方針: 白（#FFFFFF）をベースに、Slate（#64748B）系のグレーで情報の階層を作る。
 * ミニマリズム: 冗長な説明文は排除し、アイコン（Lucide React推奨）と数字、バッジ、色温度（赤・黄・緑）のみで状況を伝える。
 * 操作感: 「触れば動く」レスポンス。スマホでは「指が届く範囲」に重要アクションを配置。
2. PC版： 3×2 ダッシュボード・レイアウト
PCでは「一目で全容を把握する」ことを目的とする。
 * Grid構成: grid-cols-3 gap-6 (1280px以上)
 * Cardデザイン:
   * 背景白、境界線（border-slate-100）、角丸（rounded-xl）、控えめな影（shadow-sm）。
   * ホバー時にカード全体がわずかに浮き上がる（hover:shadow-md transition-all）。
 * ナビゲーション: 上部タブ（Sticky配置）により、スクロールしても常にカテゴリを切り替え可能にする。
3. スマホ版： モビリティと操作性
スマホでは「状況確認」と「クイックな行動」に特化する。
 * タイルレイアウト: 画像2のようなアイコン＋ラベルのタイル（grid-cols-2 または grid-cols-3）。
 * ボトムナビゲーション: 画面下部に主要機能（Home, 人材, 案件, 通知）を固定。
3.1. モバイルUXの肝： 「隠す」と「出す」
 * Floating Action Button (FAB): * 画面右下に「＋（追加）」ボタンを固定。
   * タップで「パスポートスキャン」「新規相談」などのクイックメニューを展開。
 * ボトムシート (Vaul 等を使用):
   * 詳細情報やフィルター設定は、画面遷移させずボトムシートで下からせり上げる。
   * 要件: シート外（背景）をタップすることで瞬時に閉じられる「クリック・アウト」機能を必須実装。
   * シートのヘッダーには「つまみ（Handle）」を表示し、スワイプダウンでも閉じられるようにする。
 * プログレッシブ・ディスクロージャー:
   * 最初は「氏名とスコア」だけ表示。タップすると詳細が展開される、あるいはボトムシートで詳細が表示される。

5. インタラクション要件 (AIへの指示用)
 * Feedback: ボタン押下時やデータ保存時に、Sonner (Toast) を使用して「保存完了」などを右上に表示。
 * Loading: データ取得中は、カード全体をスケルトン（Skeleton Screen）で表示し、レイアウトシフトを防ぐ。
 * Empty State: 候補者が0人の場合、「まだ登録がありません。右下のボタンから追加してください」といった、次にすべき行動をアイコンと共に中央に配置。
