# 業務ダッシュボード 要件定義書

人材紹介・派遣業者向け「AI 搭載型業務管理コックピット」の機能要件・業務要件・画面要件を定義する。技術実装の詳細は `技術要件.md`、見た目・操作感は `デザイン要件.md` を参照する。

---

## 1. システム概要

### 1.1 目的

候補者の管理から入管書類作成、収益管理までを一気通貫で行い、**管理業務の効率化**と**意思決定の補助（AI）**を両立する。

### 1.2 ターゲットユーザー

| 区分 | 説明 |
|------|------|
| 主ターゲット | 外国人（主に特定技能・技能実習）を扱う人材エージェント・派遣管理会社 |
| 利用者ロール（想定） | 営業・コーディネーター、事務（書類）、経営・管理職（数値確認） |

### 1.3 プロダクトコンセプト

- **キャッチコピー**: 「管理を便利に、判断をAIで」
- **価値提案**: カード型ダッシュボードで全体像を把握し、必要なときだけ詳細に潜る。書類作成の自動化（OCR→転記）で現場の工数を削減する。

### 1.4 スコープ（初期）

- PC・スマホ両対応のレスポンシブ Web アプリ（ブラウザ利用を前提）
- 標準機能は **3列×2行＝6枚のカード** で表現
- 業者カスタムは **最大2枚** の追加カード枠

### 1.5 スコープ外（明示的に保留するもの）

- 勤怠・給与の本番計算ロジックの完全実装（連携・取り込みは拡張枠として要件化）
- 法務・入管の最終判断の代替（AI は補助であり、人の確認が前提）

## 2. 機能要件：標準カード（3×2）

### 2.1 [A] 候補者管理・選考

#### 2.1.1 AI候補者データベース

| ID | 要件 | 詳細 |
|----|------|------|
| A-1 | 基本情報管理 | 氏名、連絡先、国籍、登録日、ステータス等を保持・一覧・検索可能とする |
| A-2 | スキル・日本語 | スキルタグ、日本語レベル（例: N5〜N1 相当）を入力・表示する |
| A-3 | AI 適性スコア | 履歴書・面談メモ等のテキストから「適性スコア」を算出・表示する（算出根拠の要約を併記できるとよい） |
| A-4 | メディア | 顔写真・添付資料の参照（権限範囲内） |

#### 2.1.2 選考・ビザ進捗パイプライン

| ID | 要件 | 詳細 |
|----|------|------|
| A-5 | ステータス管理 | 面接待ち → 内定 → ビザ申請中 → 入国待ち（必要に応じて稼働中等へ拡張）を定義する |
| A-6 | 期日・アラート | 各フェーズの想定日・期限を設定可能とし、超過時はダッシュボード上で視認できるアラートとする |
| A-7 | 通知 | アラート発生時、将来はメール/アプリ内通知と連携可能なデータ設計を想定する |

---

### 2.2 [B] 案件管理・マッチング

#### 2.2.1 クライアント・オーダー管理

| ID | 要件 | 詳細 |
|----|------|------|
| B-1 | クライアント | 派遣先（工場等）の属性、担当者、評価・特記事項を管理する |
| B-2 | オーダー | 募集人数、条件（日本語レベル、体力・職種要件等）、稼働条件、単価レンジ等を管理する |
| B-3 | 稼働状況 | 現在の稼働人数、欠員、枠の空き状況をサマリー表示する |

#### 2.2.2 AIマッチング・レコメンド

| ID | 要件 | 詳細 |
|----|------|------|
| B-4 | 照合 | オーダー条件と候補者 DB を照合し、推奨度の高い順にリスト表示する |
| B-5 | 説明 | 「なぜこの人が適しているか」を **短文（目安3行程度）** で AI が生成する |
| B-6 | 人の確認 | 採用・提案の最終決定は利用者が行う（AI は提案のみ） |

---

### 2.3 [C] 実務・収益

#### 2.3.1 書類作成オートメーション

| ID | 要件 | 詳細 |
|----|------|------|
| C-1 | OCR | パスポート等の画像から氏名・番号・生年月日等を抽出し、候補者情報へ反映する |
| C-2 | PDF 生成 | 入管関連・雇用関連など、定義したテンプレートへ自動転記した PDF を生成・保存する |
| C-3 | 多言語並記 | 雇用契約書等、母語（例: シンハラ語）との並記生成に対応する（テンプレート単位で定義） |
| C-4 | ステータス | 生成済み・翻訳済み・要修正など、書類単位のワークフロー状態を表示する |

#### 2.3.2 収益・LTVモニター

| ID | 要件 | 詳細 |
|----|------|------|
| C-5 | 売上 | 月次等の売上推移をグラフで可視化する |
| C-6 | 紹介料 | 回収ステータス（未回収・一部・完了等）を管理・表示する |
| C-7 | AI 予測 | 過去の離職データ等を踏まえ、案件・候補者単位で「利益が立ちやすい時期」等の示唆を表示する（参考情報） |
| C-8 | リスク表示 | 返金保証期間終了間近など、経営判断に効くアラートをダッシュボードで示す |

## 3. 拡張機能（カスタム枠：最大2枚）

業者ごとに **ON/OFF または差し替え** が可能な想定。

| 名称 | 概要 | 主要データ |
|------|------|------------|
| 勤怠・請求連携 | 工場からの勤怠 CSV 等を取り込み、請求額のたたき台を算出 | 取込ファイル、工場別集計 |
| 社内ナレッジ AI | 入管トラブル・過去事例をチャットで検索・要約 | ドキュメント、ベクトル索引 |

## 4. 画面要件

### 4.1 情報アーキテクチャ（上部ナビ／タブ）

| タブ | 目的 |
|------|------|
| ダッシュボード | トップの 3×2＋α の俯瞰 |
| 候補者 | 人の一覧・詳細へのショートカット |
| 案件 | クライアント・オーダー管理 |
| 実務・収益 | 書類・収益の深掘り |
| ナレッジ | 社内ナレッジ AI（拡張時） |

### 4.2 PC：ダッシュボードトップ（3×2）

各カードは **サマリー表示** と **詳細ページへの遷移** を兼ねる。

| カード | 表示内容（サマリー） | 遷移先（例） |
|--------|----------------------|----------------|
| 候補者 | 登録数、N3 以上人数、AI スコア分布、顔写真プレビュー、「今日紹介すべきトップ5」等 | 候補者一覧 |
| 進捗 | フェーズ別人数、横棒のパイプライン | 進捗管理 |
| クライアント | 工場リスト、枠の空き、稼働人数、評価 | クライアント一覧 |
| マッチング | 案件別の候補提案、理由3行 | マッチング管理 |
| 書類 | 入管・雇用の生成状態、OCR 処理数、要修正件数 | 書類管理 |
| 収益 | 月次売上、回収、LTV、離職リスクアラート | 収益管理 |
| カスタム①② | 勤怠サマリー／ナレッジ更新・チャット導線 等 | 各機能画面 |

### 4.3 スマホ：ダッシュボードトップ（タイル）

- **下部ナビ**: Home（ダッシュボード）、候補者、案件、収益・進捗、設定・その他
- **タイル（目安 10 前後）**: PC カードと対応するが、数字とアイコン中心に簡略化
- **優先アクション**: 「新規登録」「書類スキャン（OCR 開始）」等を視線の上の方に配置

### 4.4 下層：候補者一覧（PC 想定の詳細）

- **形式**: カード型リスト（写真・氏名・国籍・日本語・AI スコア・ステータス）
- **操作**: カードクリックで候補者詳細へ

### 4.5 下層：候補者詳細（タブ構成）

| タブ | 内容 |
|------|------|
| 基本情報 | 経歴、スキル、日本語、AI スコアの内訳 |
| 書類 | パスポート期限、在留カード OCR 結果、生成 PDF 一覧 |
| 派遣履歴・評価 | 派遣先、期間、クライアント評価 |
| AI Boost | 動画レジュメ解析タグ、離職リスク予測等 |

## 5. AI 機能の業務要件（プロダクト視点）

| 機能 | ユーザー価値 | 受け入れ条件の目安 |
|------|----------------|-------------------|
| OCR→転記 | 手入力削減 | アップロード後、主要フィールドが候補者に反映され、ユーザーが修正可能 |
| マッチング説明 | 提案の納得感 | 各提案に短文理由が付与される |
| ナレッジ Q&A | 調査時間短縮 | 類似事例を引用しつつ、リスクを段階（例: ％や高/中/低）で示せる |
| 収益・離職示唆 | 優先順位付け | ダッシュボードで「要フォロー」が一目で分かる |

**AI 表示**: 各エリアで「AI Boost」であることがバッジ・アイコン・短いラベルで分かること。

## 6. データ要件（業務エンティティ）

以下は論理モデル。物理スキーマは `技術要件.md` を参照。

| エンティティ | 役割 |
|--------------|------|
| 組織（管理会社） | マルチテナントの境界 |
| プロファイル | ログインユーザー（社員） |
| 候補者 | 人材マスタ＋スコア・OCR 結果 |
| クライアント | 派遣先 |
| オーダー | 求人案件 |
| 配置（プレイスメント） | 候補者×案件の進捗・結果 |
| ドキュメント | 生成 PDF・メタデータ |
| ナレッジ | 社内文書・ベクトル検索用 |

---

## 7. 非機能要件（業務観点）

| 項目 | 要件 |
|------|------|
| マルチテナント | 他社データの混在が起きないこと |
| 監査 | 重要操作（書類生成、ステータス変更）は履歴を残せる設計を推奨 |
| 可用性 | 営業時間内の利用を主とし、メンテナンス窓は別途定義 |
| 法令・個人情報 | パスポート等の取り扱いは社内規程・委託契約に準拠（保管期間・削除要請に対応可能な設計） |

---

## 8. 開発・デモの優先シナリオ

1. **3×2 ダッシュボード**で全体像を見せる  
2. **書類: OCR→自動入力→PDF** のデモで導入説得力を最大化  
3. 候補者一覧・詳細のモック→本データ接続  
4. マッチング・収益チャートの段階的統合
