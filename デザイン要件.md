# 業務ダッシュボード デザイン要件書

PC・スマホにおけるレイアウト、デザインシステム、インタラクションを定義する。機能の意味づけは `要件定義.md`、コンポーネント選定の技術面は `技術要件.md` と整合させる。

---

## 1. デザインコンセプト

### 1.1 キーワード: "Invisible Admin"（直感的な管理）

- **目的**: 管理画面であることを意識させすぎず、数字と状態で「次に何をすべきか」が伝わる UI
- **トーン**: ホワイトベースの清潔感、業務ツールとしての信頼感（遊びすぎない）
- **情報の出し方**: 冗長な説明は削り、アイコン・数値・バッジ・色の温度（成功/警告/危険）で階層化する

### 1.2 レスポンスと操作感

- 操作に対して即時の視覚フィードバック（ホバー、タップリップル、トランジション）
- スマホは **親指で届く範囲** に主要アクション（下部ナビ、FAB）

## 2. デザイントークン（カラー）

### 2.1 パレット（実装用）

技術仕様書との差異を解消し、以下を **正** とする（Tailwind の近似クラスを併記）。

| トークン | 値 | Tailwind 参考 | 用途 |
|----------|-----|----------------|------|
| `--base` | `#FFFFFF` | `white` | カード面、モーダル背景 |
| `--surface` | `#F8FAFC` | `slate-50` | ページ背景（カードとのコントラスト） |
| `--primary` | `#2563EB` | `blue-600` | プライマリボタン、リンク、選択中タブ、主要進捗 |
| `--primary-alt` | `#1E293B` | `slate-900` | 見出し強調、ナビのテキスト（信頼感のある紺） |
| `--text` | `#0F172A` | `slate-900` | 本文、重要数値 |
| `--muted` | `#64748B` | `slate-500` | 補助ラベル、メタ情報 |
| `--border` | `#F1F5F9` 〜 `#E2E8F0` | `slate-100` / `slate-200` | カード境界、区切り線 |
| `--success` | `#10B981` | `emerald-500` | 定着、書類完了、黒字寄りの状態 |
| `--warning` | `#F59E0B` | `amber-500` | 期限間近、要件ギリギリ |
| `--danger` | `#EF4444` | `red-500` | 離職リスク、返金期限、未提出・エラー |

### 2.2 AI Boost の視覚表現

- **バッジ**: 「AI」またはスパークアイコン＋短いラベル（例: 「スコア自動」「OCR済」）
- **カード内**: 通常メトリクスと区切って、淡い `surface` 背景のサブブロックに AI 由来の情報を置く
- **一貫性**: 同一のバッジスタイルを候補者・マッチング・書類・収益で再利用する

## 3. タイポグラフィ

| 用途 | 指定 |
|------|------|
| 日本語本文 | **Noto Sans JP**（Google Fonts 等で読み込み） |
| 欧文・数値 UI | **Inter**（ダッシュボードの数値・ラベルに使用可） |

**注意**: 装飾フォントはマーケ LP 向けに留め、ダッシュボード本体は可読性優先。

### 3.1 階層（目安）

- ページタイトル: 20〜24px、semibold
- カードタイトル: 14〜16px、medium
- KPI 数値: 24〜32px、bold、`--text`
- キャプション: 12px、`--muted`

## 4. PC 版レイアウト

### 4.1 グリッド

| ブレークポイント | カラム | ギャップ |
|------------------|--------|----------|
| `xl` 以上（目安 1280px+） | `grid-cols-3` | `gap-6` |
| `md`〜`lg` | `grid-cols-2` | `gap-4` |
| 狭い幅 | 1 カラム縦積み | `gap-4` |

### 4.2 カード（shadcn Card）

- 背景: `--base`
- 境界: `border`（`slate-100` 相当）
- 角丸: `rounded-xl`
- 影: 通常 `shadow-sm`、ホバー時 `shadow-md`、`transition-all`
- ホバー: わずかに持ち上がる印象（影＋任意で `translate-y-[-1px]`）

### 4.3 上部ナビゲーション

- **Sticky** ヘッダー内にタブ（ダッシュボード / 候補者 / 案件 / 実務・収益 / ナレッジ）
- スクロール時もカテゴリ切替が可能
- 選択中タブは `--primary` のボーダーまたはテキスト色で明示

### 4.4 ダッシュボードカードと KPI

- 各カード上部: アイコン＋タイトル＋任意で「詳細へ」リンク
- グラフ（パイプライン・売上）は高さを揃え、カード内で `min-h` を共通化すると整う

## 5. スマホ版レイアウト

### 5.1 タイルダッシュボード

- **3 列**または **2 列** のタイル（情報密度とタップ領域のバランスで選択）
- 各タイル: アイコン＋短いラベル＋主要 1 数値
- **優先アクション**（新規登録、パスポートスキャン）を画面上部に配置

### 5.2 ボトムナビゲーション

| スロット | ラベル例 | 役割 |
|----------|-----------|------|
| 1 | Home | ダッシュボード |
| 2 | 候補者 | 人管理 |
| 3 | 案件 | クライアント・オーダー |
| 4 | 収益/進捗 | 数値・パイプライン |
| 5 | その他 | 設定・ナレッジ入口 |

固定表示、セーフエリア（iOS）対応の `padding-bottom` を考慮。

### 5.3 Floating Action Button（FAB）

- 右下固定、「＋」またはスキャンアイコン
- タップで **スピードダイアル**: 「パスポートスキャン」「新規相談」等
- 主要フローが 1 タップで開始できること

### 5.4 ボトムシート（詳細・フィルター）

- ライブラリ: **Vaul**（shadcn 親和）を推奨
- **必須挙動**:
  - シート外（オーバーレイ）タップで **即座に閉じる**
  - 上部 **ハンドル** 表示、スワイプダウンで閉じる
- 用途: カード/タイルタップ後の詳細サマリー、一覧のフィルター、軽量フォーム

### 5.5 プログレッシブ・ディスクロージャー

- 一覧の最初の行は「氏名＋スコア」等の最小情報
- タップでボトムシートまたはアコーディオンで詳細展開

## 6. インタラクション要件

| 種別 | 仕様 |
|------|------|
| トースト | **Sonner**（shadcn 定番）で成功・失敗を右上に短時間表示 |
| ローディング | カード単位で **Skeleton**、レイアウトシフトを防ぐ |
| 空状態 | イラストまたはアイコン＋「次にすべき行動」の一文（例: 「右下から追加」） |
| エラー | `danger` 色＋再試行ボタン |
| フォーカス | キーボード操作でナビ・ボタンのフォーカスリングが見えること（WCAG を意識） |

---

## 7. チャート・データビジュアル

- パイプライン: 横棒（ステップ別人数）を優先し、色は `--primary` / `--muted` / `--warning` を段階的に使用
- 売上: 棒または折れ線、グリッド線は薄く（`slate-200`）
- アラート: カード右上に小さな `danger` ドットまたはバッジ
